User Roles

The system has four distinct roles, each with its own responsibilities and access level.

Administrator

Full system access. Creates supervisor accounts, configures rubrics and weights, manages the TW lock, and exports the final PDF report.

Supervisor

Registers FYP projects and students, submits individual teamwork grades, and assigns examiners who receive email invitations.

Examiner

External grader invited by email. Accesses a secure grading portal to evaluate the Report and/or Presentation based on their type.

Student

Evaluates teammates anonymously through the Peer Evaluation portal using a secure link generated by the system.

Admin Guide

The administrator has full control over the system. Follow these steps at the start of each semester.

1

Create Supervisor Accounts

Go to Admin Tools → Supervisor Credentials. Enter each supervisor's name, program, email, and password, then click Add Supervisor. Use Send Credentials to email login details to all new accounts at once.

The credential email includes a direct link to the system and a link to the FYP Rubrics document.
2

Configure Grading Weights

In Admin Tools → Grading Weights, set the percentage contribution for Teamwork, Report, and Presentation. Defaults are: Teamwork 35%, Report 35%, Presentation 30%. Within Teamwork: Supervisor Individual 80%, Peer Evaluation 20%.

3

Set Up the Examiner Rubric

Go to Assign Examiners → Grading Rubric. Add criteria for FYP1 and FYP2 separately for both Report and Presentation categories. Assign each criterion a max grade, weight, scope (Individual or Group), and an ABET outcome tag.

Each criterion can be linked to one ABET outcome (e.g. 1a, 3a, 5a). Outcomes appear in the results and PDF only for FYP types that have at least one criterion mapped to them — outcomes with no linked criteria are automatically hidden for that type. Outcome 2b (Feasibility Study) is always shown as Not measured for this year until its rubric criterion is added.
All criterion grades must be at least 45% of the maximum. The system enforces this automatically.
4

Configure TW Lock & Week 14 Date

In the Teamwork config panel, set the Week 14 deadline date. The system locks TW grading automatically when this date passes. You can also toggle the lock manually at any time using the TW Locked / Unlocked switch.

When the lock engages, any missing grades are automatically filled with the minimum value (45% of max grade).
5

Load & Export Final Results

Once all grading is complete, go to the Final Results tab and click Load Results. The system validates all conditions. When everything passes, Export PDF becomes active. The PDF is generated per program and per FYP type.

6

Feedback Inbox

Supervisors can send feedback or remarks through the Feedback button. An Inbox button appears in the admin header with an unread badge. Open it to read all messages.

Supervisor Guide

Supervisors manage their own projects, grade teamwork, and assign examiners.

1

Log In

Use the Supervisor ID and password sent to your email by the admin. Visit the system URL and enter your credentials on the login screen.

After logging in, check the My Tasks tab to see all pending actions — unsubmitted TW grades, pending examiner assignments, and direct links to any open grading portals.
2

Register Your FYP Projects

Go to the Registered Projects tab. Click Add New Project, enter the project title, select the type (FYP1 or FYP2), and add all student names and IDs.

Student IDs are used for peer evaluation login. Double-check them before saving.
3

Submit Teamwork Individual Grades

In the Teamwork Grading tab, select your project. Grade each student across all criteria using the Individual Assessment Matrix. Click Save Grades. The table is locked at Week 14 or by admin toggle — missing grades are auto-filled at 45%.

No grade can be lower than 45% of the criterion's maximum. The system will flag violations before saving.
4

Assign Examiners

Go to the Assign Examiners tab. Select your project, optionally add the OneDrive report link, then add examiner rows. For each examiner choose a type, then select from the supervisor list or enter the examiner's name and email.

Rules: You cannot assign yourself as an examiner. At least one Internal examiner must be assigned. Industry-only assignments are not accepted because the Report grade would be missing.
5

Send Examiner Emails

After assigning, check the Examiner Status table. Click Send All Pending Emails to invite all new examiners at once, or use the individual email button per row. Examiners with "Email Sent" status have been notified.

Results cannot be loaded if any assigned examiner has not received an invitation email.
6

Send Feedback to Admin

Use the Feedback button in the top navigation to send comments or remarks to the system administrator at any time.

Examiner Guide

Examiners receive an email invitation and access a private grading portal — no login account required.

1

Receive Invitation Email

You will receive an email containing the project title, your examiner type, and a secure personal link to the grading portal. Internal and External examiners also receive a link to the project report.

2

Open the Grading Portal

Click Open Grading Portal in the email. The link is unique to you and gives access only to your assigned project. No password is needed.

Keep your link private. Do not share it with others.
3

Grade the Report (Internal & External Only)

Review the project report using the link provided, then complete all Report rubric criteria. Use the View Report Rubric reference image to understand grading bands. Click Submit Report Grades.

4

Grade the Presentation (All Examiner Types)

After the student presentation, complete the Presentation rubric criteria. Use the View Presentation Rubric reference image. Click Submit Presentation Grades.

Grades can be saved partially and submitted later. Each section is submitted independently.
After each submission (Report and/or Presentation), a confirmation email is automatically sent to you confirming that your grades were recorded successfully.

Student Guide — Peer Evaluation

Students evaluate their teammates anonymously. No account is needed — just your Student ID.

1

Access the Peer Evaluation Portal

Open the Peer Evaluation link shared by your supervisor. The URL is: /peer.html on the system domain.

2

Enter Your Student ID

Type your Student ID (as registered in the system by your supervisor) and click Continue.

3

Rate Each Teammate

You will see a section for each of your teammates (you do not evaluate yourself). For each teammate, select a score (0–10) for every question. All fields must be filled in before submitting.

Grades are confidential. Be honest and fair — peer evaluation is part of the final Teamwork grade.
4

Submit

Click Submit Peer Evaluation. A confirmation screen confirms your submission was recorded. You can only submit once.

Grading Formula & Letter Grades

The final grade is computed from three components. Each component is expressed as a percentage (0–100).

TW Report Presentation
Individual Per Group Peer
  1. TW = Individual × 80% + Peer Evaluation × 20%
  2. Report = Average of all Internal + External examiner grades (weighted by criterion)
  3. Presentation = Average of Internal + External + Industry examiner grades (weighted by criterion)
Final Grade = TW × 35% + Report × 35% + Presentation × 30%

Conditions to Load Final Results

All of the following must be satisfied for every project before results can be loaded and the PDF exported.

All Students Submitted Peer Evaluations

Every student registered in a project must have completed and submitted their peer evaluation through the portal.

Supervisor Submitted Teamwork Grades

The supervisor must have saved the individual teamwork assessment for the project. Missing grades are auto-filled at 45% once the TW lock is activated.

At Least One Examiner Assigned

Every project must have at least one examiner assigned in the system. Projects with no examiners cannot proceed to final results.

All Examiners Have Been Emailed

Every assigned examiner must have received their invitation email (status shows "Email Sent"). Examiners with "Not Emailed" status block the results.

Internal / External Examiners Submitted Report Grades

All Internal and External examiners must have completed and submitted Report grades through their grading portal.

All Examiners Submitted Presentation Grades

All examiner types (Internal, External, and Industry) must have submitted Presentation grades. Industry examiners are only required to submit Presentation grades.

Export PDF is locked until all conditions are met

The Export PDF button remains disabled until Load Results succeeds without any incomplete warnings. A partially complete result (e.g., FYP1 ready but FYP2 not) will still block the export.

ABET Outcomes — Mapping & Computation

Each rubric criterion (Teamwork, Report, or Presentation) can be tagged with one ABET outcome. The system computes attainment per outcome per FYP type by averaging the percentage of students who scored ≥ 70% of the criterion maximum across all criteria tagged with that outcome. An outcome is only shown in results and the PDF if at least one criterion is mapped to it for that FYP type — outcomes with no linked criteria are automatically hidden.

How to Map a Criterion to an ABET Outcome

  1. Go to Assign Examiners → Grading Rubric (for Report / Presentation criteria) or Teamwork → Rubric Config (for Individual criteria).
  2. In each criterion row, find the ABET dropdown column.
  3. Select the outcome tag (e.g. 1a, 5a, 3b) that this criterion assesses. Leave it blank if the criterion is not mapped to any outcome.
  4. Save the rubric. The system will compute attainment for all mapped outcomes when final results are loaded.

The same outcome can be tagged on multiple criteria — attainment is averaged across all of them. Outcomes not tagged on any criterion for a given FYP type are hidden from results and the PDF for that type.
